## [S1] 项目概述

**项目名称**：Sky Blaster（天空突击者）
**目标用户**：7-8 岁男孩
**技术栈**：HTML5 Canvas + 原生 JavaScript（无框架依赖）
**项目位置**：`/home/sudong/mimo code/`
**视觉风格**：亮色系像素风，酷炫战机、爆炸特效、帅气技能

### 核心玩法
- 玩家控制战机在竖版卷轴关卡中自动向上飞行
- 自动射击，玩家专注于走位和躲避敌人弹幕
- 敌人从上方/两侧出现，击败后掉落经验球
- 升级时随机展示 3 个技能供选择
- 被击中后获得短暂无敌帧
- 3 个关卡，每关有多波敌人 + 阶段性 Boss

### 可扩展性设计
- 模块化架构，方便后续添加新关卡、新技能、新敌人类型
- 数据驱动的敌人/技能配置，修改 JSON 即可调整数值
- 清晰的接口定义，便于功能扩展

## [S2] 难度系统

支持 3 种难度，游戏开始前选择，影响全局数值：

| 参数 | 简单 (Easy) | 普通 (Normal) | 困难 (Hard) |
|------|-------------|---------------|-------------|
| 玩家HP | 5 | 3 | 2 |
| 无敌帧时长 | 2.0秒 | 1.5秒 | 1.0秒 |
| 敌人生成数量系数 | ×0.7 | ×1.0 | ×1.5 |
| Boss HP系数 | ×0.6 | ×1.0 | ×1.3 |
| 敌人弹速系数 | ×0.7 | ×1.0 | ×1.2 |
| 经验球掉落量 | ×1.2 | ×1.0 | ×0.8 |

## [S3] 玩家战机

### 基础属性
- **移动速度**：基础值，可通过技能提升
- **射击间隔**：基础值，可通过技能缩短
- **初始HP**：根据难度设定

### 操作方式
- **键盘**：WASD 或方向键控制移动
- **鼠标**：移动鼠标控制战机位置
- **自动射击**：战机持续自动发射子弹

### 无敌帧机制
- 被敌人或敌弹击中后获得无敌状态
- 无敌期间战机闪烁（0.2秒间隔透明/不透明）
- 无敌时长根据难度设定

---

## [S4] 技能系统

### 技能获取机制
- 击败敌人掉落经验球（蓝色菱形）
- 经验条满时触发升级
- 升级时暂停游戏，展示 3 个随机技能供选择
- 已拥有的技能再次选择时叠加升级（最多 5 级）

### 技能列表（8种）

| 技能名称 | 效果描述 | 每级提升 |
|----------|----------|----------|
| 散射弹 | 同时发射多发子弹，扇形分布 | +1 发子弹 |
| 激光束 | 发射穿透激光，伤害高 | +20% 伤害 |
| 追踪弹 | 发射自动追踪最近敌人的导弹 | +1 发导弹 |
| 护盾 | 吸收一次伤害后消失 | +1 层护盾 |
| 速度提升 | 提高战机移动速度 | +15% 速度 |
| 磁铁 | 扩大经验球吸引范围 | +30% 范围 |
| 攻击力 | 提高所有攻击伤害 | +25% 伤害 |
| 射速提升 | 缩短射击间隔 | -20% 间隔 |

### 技能展示 UI
- 全屏半透明遮罩
- 3 张技能卡片并排显示
- 卡片包含：技能图标、技能名称、当前等级、效果描述
- 点击/触摸选择技能

## [S5] 敌人系统

### 普通敌人类型（4种）

| 敌人类型 | 行为模式 | HP | 特点 |
|----------|----------|-----|------|
| 小型侦察机 | 直线向下移动 | 1 | 数量多，速度快 |
| 中型战斗机 | 斜线移动或之字形 | 2 | 中等威胁 |
| 重型轰炸机 | 缓慢向下，发射弹幕 | 4 | 高威胁，弹幕密集 |
| 自爆无人机 | 直线冲向玩家，接触爆炸 | 1 | 速度快，需优先击杀 |

### 敌人生成规则
- 从屏幕上方随机位置生成
- 每波间隔根据关卡进度缩短
- 敌人类型随关卡进度解锁
- Boss 波次前有短暂提示

## [S6] Boss 系统

### 每关 Boss 设计

**关卡 1 Boss：钢铁堡垒**
- 阶段 1（HP > 50%）：发射扇形弹幕
- 阶段 2（HP ≤ 50%）：增加弹幕密度 + 召唤小型侦察机

**关卡 2 Boss：暗影猎手**
- 阶段 1：高速移动 + 激光扫射
- 阶段 2：分身 + 交叉弹幕

**关卡 3 Boss：毁灭者**
- 阶段 1：多方向弹幕
- 阶段 2：全屏弹幕雨 + 召唤护盾无人机
- 阶段 3（HP ≤ 25%）：狂暴模式，速度翻倍

### Boss 特点
- 屏幕顶部出现，血条显示在屏幕上方
- 击败 Boss 后进入下一关或胜利画面
- Boss 弹幕有规律可循，给玩家反应时间

## [S7] 关卡设计

### 关卡结构
每个关卡包含：
1. 开场动画（战机从底部飞入）
2. 5-8 波普通敌人
3. Boss 战
4. 关卡结算（显示击杀数、获得技能等）

### 关卡进度
- **关卡 1**：森林地带，绿色背景，简单敌人组合
- **关卡 2**：沙漠要塞，黄色背景，中等难度
- **关卡 3**：机械要塞，灰色背景，高难度

### 视觉差异化
- 每关有独特的背景滚动层（视差滚动）
- 不同关卡的敌人配色方案不同
- Boss 外观与关卡主题一致

## [S8] UI 系统

### 游戏 HUD
- **左上角**：玩家 HP（心形图标）
- **右上角**：当前关卡/波次
- **底部中央**：经验条
- **底部左侧**：已获得技能图标

### 菜单系统
1. **主菜单**：
   - 游戏标题（Sky Blaster）
   - 开始游戏
   - 难度选择（Easy/Normal/Hard）
   - 操作说明

2. **暂停菜单**（按 ESC 键）：
   - 继续游戏
   - 返回主菜单

3. **游戏结束画面**：
   - 胜利/失败信息
   - 统计数据（击杀数、存活时间、获得技能）
   - 重新开始 / 返回主菜单

## [S9] 美术风格

### 视觉特征
- **色调**：明亮鲜艳，适合儿童
- **像素尺寸**：32x32 或 64x64 像素精灵
- **特效**：
  - 爆炸效果（粒子系统）
  - 子弹拖尾
  - 经验球发光效果
  - 技能释放特效

### 配色方案
- **玩家战机**：蓝色系（天空感）
- **敌人**：红色/橙色系（危险感）
- **经验球**：青色/绿色（可收集感）
- **UI**：白色/浅灰色（清晰可读）
- **背景**：渐变色（每关不同主题色）

### 字体
- 像素风格字体，确保在小屏幕上清晰可读

## [S12] 技术实现要点

### 游戏循环
- 使用 `requestAnimationFrame` 实现 60fps 游戏循环
- Delta time 计算确保帧率无关的移动速度

### 碰撞检测
- AABB（轴对齐包围盒）碰撞检测
- 简单高效，适合像素风格游戏

### 对象池
- 子弹、敌人、经验球使用对象池管理
- 避免频繁创建/销毁对象，优化性能

### 视差滚动
- 背景使用多层滚动实现深度感
- 远景滚动慢，近景滚动快

### 响应式设计
- Canvas 自适应浏览器窗口大小
- 保持原始宽高比，两侧留黑边
